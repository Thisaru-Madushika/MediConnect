## 🚀 Branching & Project Structure

To maintain clean architecture and support seamless collaboration, the project features separate modules isolated into individual Git branches:

### 🔀 Active Feature Branches

#### 1. `User-Authentication-&-Sign-up` (Account Management Module)
*   **Responsibility:** Authentication, secure onboarding, and identity verification.
*   **CRUD Operations:**
    *   **Create:** User Account Registration (Sign-Up Interface).
    *   **Read:** Identity Verification & Authentication Check (Login Interface).
    *   **Update:** Security Credentials Modification (Password Reset & Recovery).
    *   **Delete:** Account Deactivation (Data Deletion/Right to be Forgotten).

#### 2. `appointment-management` (Appointment Management Module)
*   **Responsibility:** Doctor lookup, profile auditing, and consultation scheduling.
*   **CRUD Operations:**
    *   **Create:** Scheduling new appointments (Booking Details form entry).
    *   **Read:** Browsing available specialists and viewing upcoming/past visits.
    *   **Update:** Adjusting date, time, or shifting slots (Reschedule Interface).
    *   **Delete:** Canceling active appointments prior to consultation.

## 🎨 User Interface & Experience (UI/UX)

The user interfaces are fully optimized for modern mobile form factors (designed keeping the iPhone 14 Plus specifications in mind) following a clean, accessible, and reassuring aesthetic.

### Visual Identity
*   **Primary Color:** Medical Blue (`#0056b3`) — Instills trust, security, and professionalism.
*   **Secondary Color:** Clean White (`#ffffff`) — Represents clarity and hygiene.
*   **Typography:** Sans-serif (Roboto/Inter) for immediate digital readability.

### Core Screen Maps Included:
1.  **Splash & Welcome Screen** (Onboarding entry point)
2.  **Identity Gateway** (Email Sign-In, Third-Party Google Authentication)
3.  **Account Registration** (Secure user sign-up ledger)
4.  **Credential Reset Portal** (Forgot Password flow & Token confirmation)
5.  **Doctor Search & Directory** (Filtered directory by specialty)
6.  **Scheduling & Appointment Lifecycle** (Booking sheets, Reschedule, Cancellation alerts)
